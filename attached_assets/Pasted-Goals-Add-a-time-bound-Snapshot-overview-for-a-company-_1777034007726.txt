Goals





Add a time-bound Snapshot overview for a company or trend that leverages your existing synthesis pipeline (/api/analyze + SYNTHESIS_PROMPT) and can be saved/recalled.



Add a Thought Leadership destination in the sidebar that allows only you to author/publish GEO-optimized content that persists (stored in Vercel KV/DB).



Add Data Visualization and Big Cycle Decision Engine as top menu options within the Analyze Output/results area.

Key repo reality (what we’ll build on)





The app is a single-page UI in [app/page.jsx](app/page.jsx); the results experience is driven by an in-file tab system (tabs array + activeTab) and renders OverviewTab, ForceMapTab, RoadmapTab, InvestmentThesisTab.



All LLM work (dimension agents + synthesis) ultimately calls [app/api/analyze/route.js](app/api/analyze/route.js).

Data model + persistence

Thought Leadership (Vercel KV/DB)





Storage: @vercel/kv (or equivalent KV driver), keyed by:





thoughtleadership:post:<id> → full post JSON



thoughtleadership:index → sorted list of ids (or use a ZSET keyed by publishedAt)



Post schema (JSON):





id, slug, title, dek, contentMarkdown, geoKeywords[], regions[], instruments[], companies[], publishedAt, updatedAt, status (draft|published)

Snapshot layer (Vercel KV/DB)





Storage: snapshot:<subjectHash>:<asOfDate> → snapshot JSON



Snapshot schema:





subject, subjectType, asOfDate, createdAt



inputs: minimal references (selected model, optional market snapshot ids)



snapshotSynthesis: compact synthesis output designed for “fast overview”

Server/API changes

1) Admin auth (shared secret)





Add ADMIN_PUBLISH_TOKEN env var.



Admin-only routes require header x-admin-token matching ADMIN_PUBLISH_TOKEN.

2) Thought Leadership API routes

Add new Next routes:





[app/api/thought-leadership/route.js](app/api/thought-leadership/route.js)





GET: list published posts (filter/search support)



[app/api/thought-leadership/admin/route.js](app/api/thought-leadership/admin/route.js)





POST: create/update draft



PUT: publish/unpublish



DELETE: delete



All admin methods enforce x-admin-token

3) Snapshot API routes





[app/api/snapshot/route.js](app/api/snapshot/route.js)





POST: generate or fetch a stored snapshot for { subject, subjectType, asOfDate }



Internally calls /api/analyze using a new SNAPSHOT_PROMPT(...) (shorter than full synthesis; designed for time-bound overview)



Stores result to KV keyed by subject+date

4) Big Cycle Decision Engine API route





[app/api/big-cycle/route.js](app/api/big-cycle/route.js)





POST: accept { subject, subjectType, steepData, synthesis, dataSnapshot, adanosBundle }



Use the synthesizer pattern (same LLM proxy) to return JSON with:





instrumentAttributes: Precision/Impact/Circumvention/Visibility/Speed (0–10 or 1–5; we’ll standardize)



strategicUtility: one of your six classes + rationale



capacities: SizeAsymmetries/StrategicDependencies/MarketGravity/InstitutionalEffectiveness + rationale



companyPositioning: exposure channels + main takeaways



For subjectType=trend, the system prompt hardcodes “from US perspective”.
